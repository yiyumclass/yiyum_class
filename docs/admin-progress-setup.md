# 관리자 학습 현황 설정

관리자가 유효한 강의 수강권을 기준으로 회원별 진도와 최근 학습 상태를 조회하는 화면이다.

## 적용 파일

- 화면: `/admin/progress`
- 집계 로직: `src/lib/admin/learning-progress.ts`
- 데이터베이스 함수: `supabase/migrations/20260716100000_create_admin_learning_progress.sql`

## 데이터베이스 적용

Supabase SQL Editor에서 아래 마이그레이션 파일의 내용을 실행한다.

```text
supabase/migrations/20260716100000_create_admin_learning_progress.sql
```

이 프로젝트의 원격 마이그레이션 이력과 로컬 이력이 일치하지 않을 수 있으므로, 이력 확인 없이 전체 `supabase db push`를 실행하지 않는다.

함수 `public.get_admin_learning_progress()`는 다음 조건을 모두 만족하는 데이터만 반환한다.

- 로그인 사용자가 활성 관리자일 것
- 수강권 상태가 `active`이고 만료되지 않았을 것
- 연결된 강의·챕터·차시가 공개 상태일 것
- 차시에 재생할 영상이 연결되어 있을 것

일반 회원이 함수를 호출하면 관리자 권한 검사에서 차단된다.

## 지표 기준

- **전체 진도**: 공개된 전체 차시를 같은 비중으로 계산한다.
- **완료 차시**: `completed_at`이 기록된 차시다.
- **진행 중 차시**: 재생 위치를 영상 길이로 나누며, 완료 전에는 차시 진도를 최대 99%로 제한한다.
- **최근 학습**: 수강권에 포함된 차시 중 `last_watched_at`이 가장 최근인 기록이다.
- **관심 필요**: 아직 시작하지 않았거나 마지막 학습 후 14일이 지난 수강 건이다. 완료한 수강 건은 제외한다.
- **최근 30일 학습 회원**: 최근 30일 안에 학습한 중복 없는 회원 수다.

## 키 불변 계약

- `lesson_progress`는 `course_slug`와 `lesson_id`라는 텍스트 키로 강의 콘텐츠와 결합된다.
  이 테이블에는 `courses`나 `lessons`로 향하는 외래 키(FK)가 없으며, 진도 집계는 오로지
  상품 slug와 lesson_key(차시 키) 문자열이 일치하는지로 콘텐츠와 연결된다.
- 따라서 상품 slug와 lesson_key는 생성 시점에만 입력받고 수정 UI가 없는 불변 계약이다.
  생성 후 이 값을 바꾸면 기존 `lesson_progress` 행과의 문자열 결합이 끊겨 진도가 유실된 것처럼 보인다.
- slug나 lesson_key를 바꿔야 한다면, 단순 수정이 아니라 기존 진도 데이터를 새 키로 옮기는
  이관 마이그레이션을 함께 진행해야 한다.

## 화면 확인

1. 관리자 계정으로 로그인한다.
2. 관리자 메뉴에서 **학습 현황**을 연다.
3. 강의별 평균 진도와 회원별 진도 행이 표시되는지 확인한다.
4. 상태·강의 필터, 회원/강의 검색, 정렬을 각각 확인한다.
5. 일반 회원으로 데이터베이스 함수를 직접 호출할 수 없는지 확인한다.

아직 유효한 강의 수강권이 없으면 빈 상태가 정상적으로 표시된다. 데이터베이스 함수를 적용하지 않은 환경에서는 실행할 마이그레이션 파일명이 안내된다.
