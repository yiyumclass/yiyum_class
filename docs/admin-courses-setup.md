# 어드민 3단계: 강의 관리 설정

> 이 문서는 개발·배포 담당자용이다. 실제 운영 관리자는 SQL을 실행하지 않으며,
> `상품 등록 → 강의 만들기 → 챕터·차시 추가`를 관리자 화면에서만 수행한다.
> 마이그레이션은 서비스 배포 전에 개발자가 한 번 적용한다.

## 구현 범위

- `/admin/courses` 강의 목록과 커리큘럼 편집 화면
- 강의 판매 상품과 학습 콘텐츠 1:1 연결
- 상품 관리에서 새로 만든 VOD 상품을 `강의 연결 대기` 목록에 즉시 표시
- 강의 기본 정보와 `작성 중 · 공개 · 보관` 상태 관리
- 챕터 등록·수정·위아래 순서 변경
- 차시 등록·수정·위아래 순서 변경
- 차시 추가 화면에서 MP4 선택 후 차시 생성과 영상 업로드를 한 흐름으로 처리
- 업로드 이어 올리기·교체·삭제
- 비공개 Supabase Storage와 만료되는 재생 주소 연결
- 영상 재생 시간 자동 반영, 미리보기와 차시 공개 상태 관리
- 영상 미연결 차시의 공개 차단
- 판매용 전체 커리큘럼과 실제 재생 가능한 강의실 콘텐츠 분리
- 강의·챕터·차시 변경 감사 로그

가격, 판매 상태와 이용 기간은 계속 `/admin/products`에서 관리한다. 강의 관리에서는
학습 콘텐츠만 다뤄 판매 정보와 커리큘럼의 책임을 섞지 않는다.

## 1. 강의 테이블 생성

Supabase SQL Editor에서 아래 파일 전체를 실행한다.

`supabase/migrations/20260714130000_create_courses.sql`

이 SQL은 상품 테이블과 관리자 기반이 이미 있다는 전제이므로 다음 순서를 지킨다.

1. `20260714090000_create_admin_foundation.sql`
2. `20260714110000_create_products.sql`
3. `20260714120000_expand_product_audit_log.sql`
4. `20260714130000_create_courses.sql`
5. `20260714140000_create_course_video_storage.sql`
6. `20260714150000_publish_course_catalog.sql`
7. `20260715140000_create_public_course_outline.sql`
8. `20260715150000_create_free_enrollments.sql`

강의 SQL은 다음 작업을 함께 수행한다.

- `courses`, `course_sections`, `lessons` 테이블 생성
- 활성 관리자만 조회·등록·수정할 수 있는 RLS 적용
- 챕터·차시 순서 교환 함수 생성
- 현재 SNS 수익화 클래스의 5개 챕터와 32개 차시 초기 등록
- 현재 테스트 영상 2개만 `영상 연결 · 공개` 상태로 등록
- 영상이 없는 나머지 차시는 `작성 중`으로 등록

마지막 영상 Storage SQL은 다음 작업을 수행한다.

- 비공개 `course-videos` 버킷 생성
- 관리자 전용 업로드·교체·삭제 정책 적용
- 로그인 회원에게 공개 차시만 재생할 수 있는 정책 적용
- 차시에 원본 파일명, 용량, 형식과 업로드 시각 저장
- 원본 Storage 경로 대신 만료되는 재생 주소를 발급할 수 있는 manifest 함수 생성

공개 카탈로그 SQL은 다음 작업을 수행한다.

- 공개 역할에 강의·챕터·차시 읽기 권한 부여
- `판매 중 상품 → 공개 강의 → 공개 챕터 → 공개 차시` 조건의 RLS 적용
- 작성 중·보관 콘텐츠가 수강생에게 재생되지 않도록 차단

마지막 공개 커리큘럼 개요 SQL은 다음 작업을 수행한다.

- 판매 중인 상품에 연결된 보관되지 않은 챕터·차시 제목을 판매 페이지에 제공
- 영상 경로와 내부 식별자는 제외하고 판매 화면에 필요한 정보만 반환
- 작성 중 차시는 판매 페이지의 전체 목차에는 포함하되 강의실에서는 `추후 공개`로 처리
- 보관한 강의·챕터·차시는 판매 페이지와 강의실 모두에서 제외

## 2. 화면 확인

관리자 계정으로 `/admin/courses`에 접속한다.

- 강의 정보가 정상적으로 표시되고 저장 버튼과 순서 변경 버튼이 활성화되어야 한다.
- 상품 관리에서 `VOD 강의` 유형 상품을 추가하면 `강의 연결 대기`에 표시되어야 한다.
- 연결 대기 상품의 `강의 만들기`를 누르면 해당 상품이 선택된 강의 등록 창이 열려야 한다.
- 전체 강의 1개, 챕터 5개, 차시 32개, 영상 연결 2/32가 표시되어야 한다.
- 첫 챕터를 열면 차시별 영상 연결 여부와 공개 상태를 확인할 수 있어야 한다.
- `차시 추가`에서 MP4를 선택하면 차시 생성 직후 업로드가 자동으로 시작되고 진행률과 취소 기능이 표시되어야 한다.
- 영상 없이 차시만 초안으로 추가한 경우에는 차시 목록의 `영상 업로드`에서 나중에 연결할 수 있어야 한다.
- 업로드를 마치면 영상 길이가 자동으로 반영되고 `영상 관리`로 버튼이 변경되어야 한다.
- 영상 없이 차시를 `공개`로 변경하면 서버 검증 오류가 표시되어야 한다.
- 강의를 `공개`로 변경할 때 공개 차시 중 영상 미연결 차시가 있으면 저장이 차단되어야 한다.
- 작성 중 차시의 영상 미연결은 강의 전체 공개를 막지 않아야 한다.
- `관리자 미리보기`는 작성 중 강의와 차시도 포함해 열리고 미리보기 진도는 저장되지 않아야 한다.
- 강의 상단에서 판매 페이지, 판매용 커리큘럼, 수강생 강의실의 최종 노출 상태와 차단 사유를 확인할 수 있어야 한다.

SQL 실행 전에는 현재 코드 커리큘럼을 읽기 전용으로 보여준다. 이 상태에서는 실제
저장과 순서 변경을 할 수 없다.

## 상태 운영 원칙

| 상태 | 판매 페이지 목차 | 수강생 강의실 |
| --- | --- | --- |
| `draft` | 표시 | `추후 공개`, 재생 불가 |
| `published` | 표시 | 재생 가능 |
| `archived` | 숨김 | 숨김 |

- 차시 키는 진도 기록과 연결되므로 생성 후 변경하지 않는다.
- 챕터 키와 강의 slug도 연결 키로 사용하므로 생성 후 변경하지 않는다.
- 콘텐츠는 실제 삭제하지 않고 `archived`로 보관한다.
- 운영 관리자는 영상 경로나 URL을 입력하지 않는다. 파일 선택 이후 내부 Storage 경로는
  시스템이 자동으로 생성한다.
- 무료 운영 단계에서는 MP4 파일당 50MB를 넘지 않도록 한다. Pro 전환 시 버킷과 화면의
  운영 한도를 함께 조정한다.

## 현재 사용자 화면과의 연결 범위

관리자 DB의 판매 상품과 보관되지 않은 전체 커리큘럼은 판매 화면의 단일 원본으로 사용한다.

- `/` 대표 강의명, 커리큘럼, 가격과 이용 기간
- `/courses`, `/courses/[slug]`, `/checkout`
- `/learn/[courseSlug]` 전체 목차와 공개 차시의 재생·진도 저장 대상
- `/my` 강의명과 현재 공개된 차시 기준 진도

판매 페이지에는 보관되지 않은 전체 목차가 표시된다. 강의실에서는 그중 공개된 차시만
재생할 수 있고 작성 중 차시는 `추후 공개`로 표시한다.

영상은 로그인 사용자의 유효한 상품 이용권을 서버에서 확인한 뒤 만료되는 signed URL로
교체한다. 원본 Storage 경로는 강의실에 전달하지 않는다. 결제 연동 전에는 0원 신청으로
이용권을 즉시 발급한다.

무료 미리보기 재생은 결제·체험 정책과 함께 구현하기 전까지 제공하지 않는다. 현재 차시
영상은 관리자 미리보기 또는 유효한 수강권이 있는 강의실에서만 재생한다.

배포 후 `npm run verify:admin`을 실행하면 공개 커리큘럼 RPC, 판매 중 상품 0원 정책과
무료 신청 RPC 배포 여부를 읽기 전용으로 점검할 수 있다.
