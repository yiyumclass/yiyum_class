# 어드민 2단계: 상품 관리 설정

> 이 문서는 개발·배포 담당자용이다. 실제 운영 관리자는 SQL을 실행하지 않으며,
> 관리자 화면에서 상품 등록과 수정만 수행한다. 마이그레이션은 서비스 배포 전에
> 개발자가 한 번 적용한다.

## 구현 범위

- `/admin/products` 상품 목록, 유형·상태 필터와 검색
- 강의·전자책 공통 상품 등록
- 등록 상품의 상품명·설명·판매가·이용 기간·상태 수정
- `작성 중`, `판매 중`, `판매 중지`, `보관` 상태 관리
- 기간제 또는 기간 제한 없는 이용 기간 설정
- 관리자 권한을 다시 확인하는 Server Action과 DB RLS
- 상품 등록·수정·상태 변경 시 `admin_audit_logs` 자동 기록

상품은 판매 단위만 관리한다. 강의의 챕터·차시와 전자책 파일은 다음 단계에서
`courses`, `course_sections`, `lessons`, `ebooks`로 분리해 연결한다.

## 1. 상품 테이블 생성

Supabase SQL Editor에서 아래 파일 전체를 실행한다.

`supabase/migrations/20260714110000_create_products.sql`

이 마이그레이션은 다음 작업을 함께 수행한다.

- `public.products` 생성
- 현재 `이윰 SNS 수익화 클래스` 상품 초기 등록
- 판매 중 상품은 공개 조회 허용
- 활성 관리자만 전체 조회·등록·수정 허용
- 실제 삭제 권한은 제공하지 않고 `archived` 상태 사용
- 변경 관리자와 이전·변경 상태를 감사 로그에 기록

기본 상품 테이블 SQL을 이미 실행했다면, 변경 필드와 변경 전후 값을 더 자세히
기록하도록 아래 파일도 이어서 실행한다.

`supabase/migrations/20260714120000_expand_product_audit_log.sql`

결제 연동 전 무료 신청 테스트를 사용하려면 강의 마이그레이션 적용 후 아래 파일도
실행한다.

`supabase/migrations/20260715150000_create_free_enrollments.sql`

이 마이그레이션은 다음 작업을 수행한다.

- 강의·전자책 상품 가격을 0원으로 변경
- 테스트용 전자책 상품을 판매 중·0원 상태로 등록
- 사용자별 강의·전자책 이용권과 본인 조회 RLS 생성
- 로그인 사용자의 0원 상품 즉시 신청 RPC 생성
- 신청한 사용자만 강의실 영상과 진도 저장 기능을 사용할 수 있도록 제한

## 2. 화면 확인

관리자 계정으로 다시 `/admin/products`에 접속한다.

- 상단에 `DB 연결됨`이 표시되어야 한다.
- `새 상품 등록` 버튼이 활성화되어야 한다.
- 기존 SNS 수익화 클래스가 판매 중 상태로 표시되어야 한다.
- 각 DB 상품 우측의 `수정`을 누르면 기존 가격과 상품 정보가 채워져야 한다.
- 무료 신청 모드에서는 판매가 입력이 0원 읽기 전용으로 표시되어야 한다.
- 저장 후 목록의 가격, 이용 기간, 상태와 최근 수정일이 갱신되어야 한다.

SQL 실행 전에도 화면은 열리지만, 현재 코드 카탈로그를 읽기 전용으로 보여주며
상품 등록 버튼은 비활성화된다.

## 상태 운영 원칙

| 상태 | 의미 | 공개 판매 |
| --- | --- | --- |
| `draft` | 작성·검수 중 | 아니요 |
| `active` | 판매 중 | 예 |
| `paused` | 일시 판매 중지 | 아니요 |
| `archived` | 운영 종료 후 보관 | 아니요 |

주문·수강권 기록과의 연결을 고려해 상품 행은 삭제하지 않는다. 운영 종료 상품은
`archived`로 보관한다.

## 수정 운영 원칙

- 상품명, 설명, 판매가, 이용 기간, 판매 상태, 이미지·상세 경로는 수정할 수 있다.
- 상품 유형과 상품 주소(`slug`)는 주문 및 콘텐츠 연결 키로 사용하므로 생성 후에는
  화면에서 수정하지 않는다.
- 판매가 변경은 기존 주문 금액을 바꾸지 않고, 이후 결제에 적용할 상품 가격만 바꾼다.
- 모든 변경은 Server Action에서 관리자 권한을 다시 확인하며, 감사 로그에 변경 필드와
  변경 전후 값을 남긴다.

## 현재 공개 판매 화면과의 연결 범위

`/`, `/courses`, 강의 상세와 `/checkout`은 상품 DB의 판매 상태·가격·이용 기간을 사용한다.
결제 연동 전에는 애플리케이션의 무료 신청 모드가 판매가를 0원으로 고정한다. 로그인 사용자가
무료 신청을 완료하면 서버 RPC가 이용권을 발급하고 `/my`에 해당 콘텐츠가 즉시 표시된다.

무료 신청 모드에서는 어드민 판매가도 0원 읽기 전용으로 표시한다. 운영자가 입력한 금액을
조용히 무시하지 않으며, 결제 도입 시 무료 모드를 끈 뒤 판매가 편집을 다시 활성화한다.

결제 도입 시에는 무료 신청 모드를 끄고, 결제 성공 웹훅이 동일한 이용권 테이블에
`payment` 출처로 권한을 발급하도록 연결한다.
