## 전체 진행 상황

```
[설계] ──────────────────────────────────── ✅ 문서화 완료
[인증]
  ├─ Supabase 연동 (클라이언트/서버/미들웨어) ── ✅ 코드 완료
  ├─ 카카오 로그인 ─────────────────────────── ⏸ 비즈앱 전환 대기 (KOE205)
  └─ 전화번호 본인인증 ────────────────────── ⬜ 미착수
[상품/스키마]
  ├─ products/orders/entitlements 테이블 ───── ⬜ 미착수
  └─ RLS 정책 ──────────────────────────────── ⬜ 미착수
[결제]
  ├─ 토스 결제위젯 (테스트 키) ──────────────── ⬜ 미착수
  ├─ 주문 생성 API ─────────────────────────── ⬜ 미착수
  └─ 결제 승인 API (금액검증+승인+이용권) ──── ⬜ 미착수
[접근 제어]
  └─ entitlement 기반 콘텐츠 게이팅 ────────── ⬜ 미착수
[라이브 전환]
  └─ 토스 계약 + 실 키 교체 ────────────────── ⬜ 미착수
```

## 핵심 결정 사항 (요약)

- **가입과 결제 분리**: 가입은 무료, 강의·전자책마다 개별 결제 → 이용권(entitlement) 부여.
- **DB/인증**: Supabase (Auth + Postgres + RLS).
- **결제**: 토스페이먼츠 일회성 결제. 공개 테스트 키로 계정 없이 전 구간 테스트 가능.
- **인증 방식**: 카카오 로그인 우선 (비즈앱 전환 대기), 필요 시 이메일+비번 대안.
- **로그인 게이트**: 랜딩은 오픈, "결제하려는 순간"에만 로그인 요구 (로그인 후 원래 흐름 복귀).
